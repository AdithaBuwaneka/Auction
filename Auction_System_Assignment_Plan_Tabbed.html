‚è±Ô∏è Auction Logic & Timing Mechanism

Auction Rules

  • Seller Configuration: Sets START time, END time (mandatory), and BID GAP
  • Bid Gap: Time window after each bid for next bid to be placed
  • Mandatory End: Auction MUST end by this time regardless of bid activity
  • Early Closure: If no bid within BID GAP, auction ends automatically
  • Deadline Extension: Each new bid extends deadline by BID GAP (up to mandatory end)

Race Condition Prevention

‚ö†Ô∏è The Problem

Without proper synchronization, two users could bid at the same time and the lower bid could win because both threads read the same current price before either updates it.

‚úÖ The Solution

Use synchronized blocks to ensure only one thread can validate and place a bid at a time. This guarantees the highest bid always wins.

ByteBuffer Concept

ByteBuffer States

ByteBuffer is used for efficient data transfer with three key attributes:

  • position: Next byte to read/write
  • limit: First byte that should not be read/written
  • capacity: Maximum size of the buffer

Key Operations:

  • flip(): Switch from write mode to read mode
  • clear(): Reset for next write operation
  • remaining(): Get number of bytes available to read
